# ML Fraud Detection. Версия 2

Проект построения системы обнаружения мошеннических транзакций на основе потоковых данных с использованием Kafka, PostgreSQL, Streamlit UI и ML-модели (XGBoost). Проект реализует полный цикл обработки данных от загрузки до визуализации результатов.

Данные для работы с сервисом доступны по ссылке: [Teta ML 1 2025](https://www.kaggle.com/competitions/teta-ml-1-2025).

## Архитектура

Проект состоит из следующих сервисов:

| Сервис           | Описание |
|------------------|----------|
| **Kafka**        | Шина сообщений для потоковой передачи данных между компонентами |
| **Zookeeper**    | Управление кластером Kafka |
| **Kafka UI**     | Веб-интерфейс для мониторинга Kafka |
| **fraud_detector** | Микросервис, выполняющий предобработку данных и инференс модели XGBoost |
| **scoring_writer** | Потребитель Kafka, сохраняет результаты скоринга в PostgreSQL |
| **PostgreSQL**   | Хранилище результатов скоринга |
| **interface**    | Веб-интерфейс (Streamlit) для загрузки файлов и просмотра результатов |
| **Prometheus**   | Система мониторинга и сбора метрик |
| **Grafana**      | Визуализация метрик и создание дашбордов |
| **Node Exporter**| Сбор метрик о состоянии системы (CPU, память и т.д.) |

## Функциональность

### 1. Загрузка CSV-файла с транзакциями
- Поддерживается только формат `.csv`.
- Каждая строка отправляется в Kafka топик `transactions` с уникальным `transaction_id`.

### 2. Обработка и скоринг транзакций
- Сервис `fraud_detector` считывает данные из топика `transactions`.
- Выполняет препроцессинг и предсказание с помощью модели XGBoost.
- Результат (`score`, `fraud_flag`) отправляется в топик `scoring`.

### 3. Хранение результатов
- Сервис `scoring_writer` читает топик `scoring`.
- Сохраняет результаты в PostgreSQL таблицу `scores`.

### 4. Визуализация результатов
Через кнопку "Посмотреть результаты" можно:
  - Посмотреть **10 последних фродовых транзакций** (`fraud_flag == 1`)
  - Посмотреть **гистограмму распределения скоров** по последним 100 транзакциям

### 5. Мониторинг и метрики
- Система собирает метрики о работе сервисов и железа
- Доступны дашборды для визуализации:
  - Метрик модели (скоры предсказания, распределение скоров)
  - Групповых метрик скоринга: фильтрация по штату и продавцу, доля фрода по категориям товара
  - Производительности (время обработки транзакций, скорость записи в БД)
  - Системных ресурсов (CPU, RAM, диск, сеть)

## Как запустить проект

### 1. Клонируйте репозиторий

```bash
git clone https://github.com/karakumka/mlops_hw2.git
```

### 2. Создайте `.env` файл

Скопируйте шаблон `.env.example` в `.env`:

```bash
cp .env.example .env
```

### 3. Запустите контейнеры

```bash
docker-compose up --build
```
Убедитесь, что у вас установлены:
   - [Docker](https://docs.docker.com/engine/install/)
   - [Docker Compose](https://docs.docker.com/compose/install/)

Первый запуск может занять несколько минут — образы будут скачаны и собраны.

## Мониторинг и метрики

Проект включает мониторинг и визуализацию с помощью Prometheus и Grafana. Prometheus собирает технические и прикладные метрики сервисов, а Grafana объединяет их с данными из PostgreSQL для построения аналитических графиков по результатам скоринга.

Доступны следующие метрики:

### Метрики системы (Node Exporter)
- Использование CPU
- Использование памяти
- Нагрузка на диск
- Сетевая активность

### Метрики приложения
- Количество обработанных транзакций
- Распределение скоров мошенничества
- Время обработки транзакций
- Доля мошеннических транзакций

### Групповые метрики скоринга
- Фильтрация графиков по штату (`us_state`)
- Фильтрация графиков по продавцу (`merch`)
- Плотность распределения скоров и TPS обработки транзакций для выбранных групп
- Доля фродовых транзакций по категории товара (`cat_id`) на последних 1000 транзакциях

Для доступа к дашбордам Grafana используйте:
- URL: http://localhost:3000
- Логин: admin
- Пароль: admin

## Таблица в PostgreSQL

Таблица `scores` содержит следующие поля:

| Поле             | Тип          | Описание                                   |
|------------------|--------------|--------------------------------------------|
| id               | SERIAL       | Уникальный ID                              |
| transaction_id   | TEXT         | Идентификатор транзакции                   |
| score            | FLOAT        | Вероятность мошенничества                  |
| fraud_flag       | INT          | 1 — мошенничество, 0 — норма (порог = 0.97)|
| us_state         | TEXT         | Двухбуквенный идентификатор штата          |
| merch            | TEXT         | Наименование продавца                      |
| cat_id           | TEXT         | Категория товара                           |
| created_at       | TIMESTAMP    | Дата и время записи                        |

## Как использовать

1. Перейдите в браузере по адресу: [http://localhost:8501](http://localhost:8501)
2. Загрузите CSV файл с данными транзакций.
3. Нажмите "Отправить", чтобы данные попали в Kafka.
4. После обработки можно нажать "Посмотреть результаты":
- Отобразятся последние 10 фродовых транзакций
- Будет показана гистограмма скоров последних 100 транзакций
5. Для просмотра метрик откройте Grafana [http://localhost:3000](http://localhost:3000)

### Очистка после тестирования

Остановите и удалите контейнеры:
```bash
docker-compose down -v
```
Если нужно очистить данные и метрики:
```bash
docker volume rm $(docker volume ls -q | grep -E 'postgres_data|prometheus_data|grafana_data')
```
